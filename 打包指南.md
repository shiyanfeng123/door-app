# 移动应用打包指南

本指南将帮助您将 React 项目打包为可安装在安卓和苹果手机上的应用。

## 步骤 1：解决 PowerShell 执行策略问题

如果您在运行 npm 或 npx 命令时遇到执行策略错误，请按照以下步骤解决：

1. 以管理员身份打开 PowerShell
2. 运行命令：`Set-ExecutionPolicy RemoteSigned -Scope CurrentUser`
3. 输入 `Y` 确认更改

## 步骤 2：安装 Capacitor 依赖

在项目根目录执行以下命令：

```bash
npm install @capacitor/core @capacitor/cli
```

## 步骤 3：初始化 Capacitor（如果尚未初始化）

如果您的项目还没有 Capacitor 配置，请运行：

```bash
npx cap init
```

按照提示输入应用信息，或者使用已有的 `capacitor.config.json` 配置。

## 步骤 4：添加移动平台

### 添加 Android 平台

```bash
npx cap add android
```

### 添加 iOS 平台

```bash
npx cap add ios
```

## 步骤 5：构建 Web 应用

```bash
npm run build
```

## 步骤 6：同步代码到移动平台

```bash
npx cap sync
```

## 步骤 7：构建和打包应用

### Android 打包

1. 打开 Android 项目：
   ```bash
   npx cap open android
   ```

2. 在 Android Studio 中：
   - 选择 "Build" > "Generate Signed Bundle / APK"
   - 按照向导创建签名密钥（如果没有）
   - 选择 "APK" 选项
   - 完成构建过程

### iOS 打包

1. 打开 iOS 项目：
   ```bash
   npx cap open ios
   ```

2. 在 Xcode 中：
   - 选择 "Product" > "Archive"
   - 完成归档过程
   - 在 "Organizer" 中选择构建的归档
   - 点击 "Distribute App" 并按照向导完成打包

## 注意事项

1. **iOS 打包要求**：
   - 需要 macOS 系统
   - 需要安装 Xcode
   - 需要 Apple Developer 账号

2. **Android 打包要求**：
   - 需要安装 Android Studio
   - 需要配置 JDK 和 Android SDK

3. **签名密钥**：
   - 为了发布应用，您需要创建签名密钥
   - 请妥善保管签名密钥，丢失后无法恢复

4. **权限配置**：
   - 根据应用需要，在 `AndroidManifest.xml` 和 `Info.plist` 中配置相应的权限

## 故障排除

如果遇到问题，请检查：

1. 确保所有依赖已正确安装
2. 确保 Web 应用构建成功
3. 确保 Capacitor 配置正确
4. 查看 Capacitor 官方文档获取更多帮助

祝您打包成功！
